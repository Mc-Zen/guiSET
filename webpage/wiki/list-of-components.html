    Control    (reference)

    • Base Component from which you can't create an instance itself (maybe you're familiar with "abstract" classes) but every other component inherits from Control.
    • Provides most of the event and rendering functionality.
    • Events: press, release, move, drag, enter, exit, wheel, focus, resize

    Container     (reference)

    • A component for grouping elements.
    • Containers can be arbitrarily nested.
    • This is the simplest container which will place its content items at the x- and y-position specified for the individual items (relative to top left corner of the container).
    • Content items may overlap depending on how you set their position. You have control on which items to place above others through the z property.

    Frame     (reference)

    • Master container (inherits Container).
    • Provides methods to change window icon, title, resizable...
    • Under the hood deals with the rendering, shortcuts, events, ...

    Button     (reference)

    • Simple push button. Background color changes automatically when mouse hovers over button or presses it. Both hoverColor and pressedColor can be set manually.
    • Has a constructor for setting text and pressEvent method immediately: Button(String text, String methodname).
    • Provides constructor for initializing with background image.

    Label     (reference)

    • Basic label to display single or multiline text. Default align is CENTER.
    • Provides constructor for initializing with background image

    VFlowContainer     (reference)

    • Added items are automatically placed one below the other. Ignores coordinates of items. Paddings and margins are honoured,

    VScrollContainer     (reference)

    • Like VFlowContainer but when the content does not fit the Container height it becomes scrollable.
    • Scroll speed can be set to any value (negative values will result in reversed scroll behavior).
    • There is an option for a slim scroll handle that's more like the scroll position indicators on mobile phones.

    HFlowContainer     (reference)

    • Like VFlowContainer but horizontal.

    HScrollContainer     (reference)

    • Like VScrollContainer but horizontal.

    FlowContainer     (reference)

    • This works like the other flow containers but the items will be arranged from left to right and break to new lines from top to bottom when the width is exceeded.

    ScrollArea     (reference)

    • More like the plain Container, relative positioning according to the items x and y coordinate but allows scrolling vertically and horizontally if dimensions are exceeded.

    Checkbox     (reference)

    • A simple checkbox that can be ticked and unticked. The text (if set) is drawn to the right of the checkbox.
    • Events: checkChange

    Switch     (reference)

    • Version of Checkbox with different graphics: looks like an android switch and toggeling between on/off is animated.
    • Events: checkChange

    Slider     (reference)

    • A floating point value slider. User can set value by clicking, pulling or scrolling. Minimum and maximum can be set freely. Orientation can be set to Slider.HORIZONTAL or Slider.VERTICAL with slider.setOrientation(int orientation).
    • Events: valueChange

    Knob     (reference)

    • A floating point value knob. User can set value by clicking, pulling or scrolling. Minimum and maximum can be set freely. Drag orientation can be set to Knob.HORIZONTAL or Knob.VERTICAL with knob.setOrientation(int orientation).
    • Events: valueChange

    Textbox     (reference)

    • Single-line text input field. Scrollable. Features text selection and copy/cut/paste.
    • Events: textChange, keyPress, submit

    PasswordTextbox     (reference)

    • Subclass of textbox where all chars are displayed as the passwordCharacter and copying via Ctrl+C/Ctrl+X is forbidden.

    MultilineTextbox     (reference)

    • Textbox that breaks overflowing lines and allows multi-line input. Scrollable. Features text selection and copy/cut/paste.
    • Events: textChange, keyPress

    ListView     (reference)

    • Create a list of elements - a bit similar to VScrollContainer but this one provides an itemSelected event, fired when items are selected (obviously). Multiple selections can be enabled and a color for selected items specified. You can simply add text items by calling listview.add(String text). This creates a new ListItem and appends it to the view. You can also create and style your own list items (or subclass ListItem) and even add other controls like checkboxes although they will not fire the itemSelect event and might not look "selected" if they are.
    • Events: itemSelect

    ListItem     (reference)

    • Look at ListView.

    MenuBar     (reference)

    • Container for menus.
    • More info about creating menus in wiki article about menus.

    MenuItem     (reference)

    • More info about creating menus in wiki article about menus.
    • Events: select, itemSelect

    MenuTextbox     (reference)

    ContextMenu     (reference)

    • Can be used to create menu strips that are independent from the main menu, such as appear on right click on some area.
