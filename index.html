Welcome to guiSET.

(For the impatient: Getting started, source code on GitHub)

Intention

This library is intended for creating advanced graphical user interfaces in Processing. You can write Processing sketches entirely with an empty draw() loop, meaning using only this library to create the user surface. Or you can combine both: responsive UI elements with the traditional simplicity-oriented drawing in Processing.

What is this

guiSET is a large library to include proper graphical user interfaces in Processing sketches. It features quite a few classes to create buttons, textboxes, lists, nested containers and many more (for a complete list look here). The main goals of guiSET are:

  • Highly customizable styles
  • Event listeners and a lot of functionality
  • Fairly simple and intuitive
If you're new to guiSET, take a look at the introduction and the examples. Further elaborations and the main concept can be found on this page if you want to develop own components or contribute to the library.
